# Docker: Краткое руководство

## I. Основы

* **Docker:** Платформа для разработки, доставки и запуска приложений в контейнерах.
* **Контейнеризация:** Форма виртуализации на уровне операционной системы. Изоляция приложений и их зависимостей.
* **Виртуальные машины (VM):** Эмулируют целую операционную систему. Более тяжеловесны, чем контейнеры.
* **Образ (Image):** Шаблон только для чтения, используемый для создания контейнеров. Содержит код, runtime, системные инструменты, библиотеки и настройки.
* **Контейнер (Container):** Запущенный экземпляр образа. Изолирован от других контейнеров и хост-системы.
* **Dockerfile:** Текстовый файл, содержащий инструкции по сборке образа.
* **Docker Hub:** Публичный реестр образов Docker.
* **Docker Registry:** Место хранения Docker образов (может быть публичным или приватным).

## II. Основные Команды

* **`docker pull <image_name>`:** Скачивает образ из реестра.
* **`docker images`:** Список локально сохраненных образов.
* **`docker run <image_name>`:** Создает и запускает контейнер из образа.
* **`docker ps`:** Список запущенных контейнеров.
* **`docker ps -a`:** Список всех контейнеров (включая остановленные).
* **`docker stop <container_id>`:** Останавливает контейнер.
* **`docker start <container_id>`:** Запускает остановленный контейнер.
* **`docker restart <container_id>`:** Перезапускает контейнер.
* **`docker rm <container_id>`:** Удаляет контейнер.
* **`docker rmi <image_id>`:** Удаляет образ.
* **`docker build -t <image_name> .`:** Собирает образ из Dockerfile в текущем каталоге.
* **`docker exec -it <container_id> bash`:** Запускает shell-сессию внутри контейнера.
* **`docker logs <container_id>`:** Отображает логи контейнера.
* **`docker inspect <container_id>`:** Предоставляет подробную информацию о контейнере.
* **`docker top <container_id>`:** Отображает процессы, работающие внутри контейнера.
* **`docker port <container_id>`:** Отображает назначенные порты контейнера.

## III. Dockerfile Инструкции

* **`FROM <image_name>`:** Базовый образ для сборки.
* **`RUN <command>`:** Выполняет команду внутри образа.
* **`COPY <src> <dest>`:** Копирует файлы из хост-системы в образ.
* **`ADD <src> <dest>`:** Аналогично `COPY`, но может распаковывать архивы и скачивать файлы из URL.
* **`WORKDIR <directory>`:** Устанавливает рабочий каталог внутри образа.
* **`EXPOSE <port>`:** Объявляет, что контейнер прослушивает определенный порт.
* **`ENV <key> <value>`:** Устанавливает переменные окружения.
* **`CMD ["executable","param1","param2"]`:** Задает команду для запуска контейнера. Может быть переопределена при запуске.
* **`ENTRYPOINT ["executable","param1","param2"]`:** Задает основную команду для запуска контейнера. Не переопределяется при запуске (можно добавить аргументы).
* **`VOLUME <mount_point>`:** Создает точку монтирования для постоянного хранения данных.
* **`USER <user>`:** Устанавливает пользователя для запуска процессов в контейнере.

## IV. Сети и Тома

* **Docker Networks:** Позволяют контейнерам взаимодействовать друг с другом. Типы: bridge, host, none, overlay.
* **Bridge Network:** Создает виртуальную сеть, к которой подключаются контейнеры.
* **Host Network:** Контейнеры используют сетевой интерфейс хост-системы.
* **Overlay Network:** Для multi-host networking (Docker Swarm).
* **Docker Volumes:** Для сохранения данных между запусками контейнеров. Типы: named volumes, anonymous volumes, bind mounts.
* **Named Volumes:** Именованные тома, управляемые Docker.
* **Anonymous Volumes:** Тома, создаваемые Docker без имени.
* **Bind Mounts:** Монтируют файлы или директории с хост-системы в контейнер.

## V. Docker Compose

* **Docker Compose:** Инструмент для определения и запуска многоконтейнерных приложений.
* **`docker-compose.yml`:** Файл YAML, определяющий сервисы, сети и тома для приложения.
* **`docker-compose up`:** Создает и запускает сервисы, определенные в `docker-compose.yml`.
* **`docker-compose down`:** Останавливает и удаляет сервисы, определенные в `docker-compose.yml`.
* **`docker-compose ps`:** Список сервисов, определенных в `docker-compose.yml`.

## VI. Docker Swarm

* **Docker Swarm:** Инструмент для оркестрации контейнеров.
* **Manager Nodes:** Управляют кластером.
* **Worker Nodes:** Выполняют контейнеры.
* **Services:** Определяют, как запустить и масштабировать контейнеры.
* **Tasks:** Отдельные экземпляры контейнера, составляющие сервис.

## VII. Best Practices

* **Используйте `.dockerignore`:** Исключите ненужные файлы и каталоги из контекста сборки.
* **Многослойность образов:** Используйте слои Dockerfile эффективно для кэширования.
* **Минимизируйте размер образа:** Удалите ненужные пакеты и файлы.
* **Безопасность:** Используйте актуальные образы и соблюдайте принципы наименьших привилегий.
* **Логирование:** Настройте централизованное логирование.
* **Мониторинг:** Отслеживайте производительность контейнеров и хост-системы.
