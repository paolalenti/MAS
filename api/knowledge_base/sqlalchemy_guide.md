# SQLAlchemy: Краткое руководство

## I. Основы

* **SQLAlchemy:** Мощный и гибкий Python SQL Toolkit и Object Relational Mapper (ORM).
* **Core:** Низкоуровневый доступ к SQL. Позволяет вручную создавать и выполнять SQL-запросы.
* **ORM (Object Relational Mapper):** Сопоставляет объекты Python с таблицами в базе данных.
* **Declarative Base:** Основной класс для определения моделей таблиц.
* **Session:** Группа транзакций, используемых для взаимодействия с базой данных.
* **Engine:** Подключение к базе данных.

## II. Основные Компоненты

* **Table:** Определение таблицы в базе данных. Состоит из столбцов и ограничений.
* **Column:** Определение столбца в таблице. Указывает тип данных и ограничения.
* **MetaData:** Контейнер для хранения информации о таблицах и других объектах базы данных.
* **Primary Key:** Уникальный идентификатор записи в таблице.
* **Foreign Key:** Связь между таблицами.
* **Relationship:** Определяет связь между моделями Python.

## III. Основные Типы Данных

* **Integer:** Целое число.
* **Float:** Число с плавающей точкой.
* **String:** Строка.
* **Boolean:** Логическое значение.
* **Date:** Дата.
* **DateTime:** Дата и время.
* **Text:** Длинная строка.
* **Enum:** Перечисление.

## IV. Основные Операции

* **`create_all()`:** Создает таблицы в базе данных на основе моделей.
* **`session.add()`:** Добавляет объект в сессию.
* **`session.commit()`:** Сохраняет изменения в базе данных.
* **`session.rollback()`:** Отменяет изменения в базе данных.
* **`session.query()`:** Создает запрос к базе данных.
* **`filter()`:** Добавляет условия к запросу.
* **`order_by()`:** Сортирует результаты запроса.
* **`limit()`:** Ограничивает количество возвращаемых результатов.
* **`offset()`:** Пропускает определенное количество результатов.
* **`first()`:** Возвращает первый результат запроса.
* **`all()`:** Возвращает все результаты запроса.

## V. Связи (Relationships)

* **One-to-One:** Один объект связан с одним другим объектом.
* **One-to-Many:** Один объект связан с несколькими другими объектами.
* **Many-to-One:** Несколько объектов связаны с одним другим объектом.
* **Many-to-Many:** Несколько объектов связаны с несколькими другими объектами.
* **`relationship()`:** Функция для определения связей между моделями.
* **`back_populates`:** Используется для двусторонних связей.

## VI. Выражения (Expressions)

* **`select()`:** Создает выражение SELECT.
* **`insert()`:** Создает выражение INSERT.
* **`update()`:** Создает выражение UPDATE.
* **`delete()`:** Создает выражение DELETE.
* **`column()`:** Представляет столбец в таблице.
* **`literal()`:** Представляет литеральное значение.
* **`func`:** Используется для вызова встроенных функций SQL.

## VII. Явные Транзакции

* **`session.begin()`:** Начинает явную транзакцию.
* **`session.commit()`:** Подтверждает транзакцию.
* **`session.rollback()`:** Откатывает транзакцию.
* **`with session.begin():`:** Использование контекстного менеджера для автоматического управления транзакциями.

## VIII. Расширенные Концепции

* **Events:** Позволяют перехватывать события базы данных.
* **Caching:** Кэширование результатов запросов.
* **Connection Pooling:** Управление соединениями с базой данных.
* **Asynchronous SQLAlchemy:** Использование SQLAlchemy с асинхронными библиотеками.

## IX. Стили Кодирования

* **Declarative vs. Imperative:** Используйте декларативный стиль для определения моделей и связей.
* **Explicit Transactions:** Используйте явные транзакции для обеспечения целостности данных.
* **Code Readability:** Пишите чистый и понятный код.
