# Основы Docker: Краткий справочник

Docker — это платформа для разработки, доставки и запуска приложений в контейнерах.

## Основные понятия
- **Образ (Image)**: Инертный чертеж или шаблон. Это "замороженная" файловая система со всем необходимым кодом и зависимостями.
- **Контейнер (Container)**: Запущенный экземпляр образа. Если образ — это рецепт пиццы, то контейнер — это сама готовая пицца на столе.
- **Dockerfile**: Текстовый файл с инструкциями, по которым собирается образ.

## Зачем это нужно?
Docker решает проблему "на моем компьютере всё работает". Он создает изолированную среду, которая будет вести себя одинаково и на ноутбуке разработчика, и на сервере в облаке. 💡 Помни: контейнер должен быть эфемерным (его должно быть легко удалить и создать заново без потери данных).
